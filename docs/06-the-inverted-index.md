## 重點摘要 (Key Summary)

- **倒排索引的定義**：倒排索引是所有現代資訊檢索系統的核心資料結構，它利用詞項-文件矩陣的稀疏性來實現高效檢索。
- **基本結構**：對於每個詞項，儲存包含該詞的所有文件的列表（張貼列表），每個文件用唯一的文件 ID（docID）來標識。
- **字典與張貼集合**：左邊是字典（詞項及其統計資訊），右邊是張貼集合（按文件 ID 排序的文件列表）。字典通常放在記憶體中，張貼集合則可儲存在磁碟上。
- **為何使用可變大小列表**：不同詞項出現在不同數量的文件中，固定陣列無法有效處理這種變異性；動態索引也需要彈性的資料結構。
- **索引建構流程**：
  1. 文件經過分詞（tokenization）轉換成詞元序列
  2. 透過語言處理模組進行正規化（小寫轉換、詞幹還原等）
  3. 以詞項為主鍵、文件 ID 為次鍵進行排序
  4. 合併同一文件中的重複項目
  5. 將同一詞項的所有文件合併成張貼列表
- **文字前處理步驟**：包含分詞（tokenization）、正規化（normalization）、詞幹還原（stemming）以及停用詞（stop words）的處理。
- **儲存效率**：張貼列表是最大的儲存開銷，但其上限受限於集合中的詞元總數，因此儲存空間是可管理的。可透過壓縮技術進一步減少儲存需求。
